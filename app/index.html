本学ネットワークの利用規則

禁止事項に違反すると、アカウントが停止され指導が入る。 アカウントの貸し借りや、演習室内での飲食などはやってしまいがちなので気をつける必要がある。また、勿論だが禁止事項の他に 日本国の法律も十分留意して行動するように。

Macの説明

演習室にあるiMacは21.5inch/intel Core i5 2.7GHz/RAM 8GB/SSD 256GBのマシンである。記憶容量の少なさに目がついたかもしれないが、 大半のデータはTSUBAMEで保管されるので大容量ストレージが要らないのである。電源スイッチはiMacの裏にあり、裏側には他にもUSBポート等がある。 見ればわかると思うが、机の上にはキーボードとマウスもある。素晴らしい！！

東工大ポータルとその利用

東工大の各種サービスの利用には東工大ポータルにログインする必要性がある。 入学式の際に配布されたアカウント通知書にIDとパスワードが記載されている他、学生証の裏にマトリクスコードがあるのでログインを行う。 マトリクスコードの入力は面倒だが、東工大なりに工夫してセキュリティを担保するための最適解だったのだろうから仕方ないので、頑張って入力する。 ログインできたらまずパスワードを変更すること。忘れないながらも、十分に強度なパスワードを設定する。

Tokyo Tech Web Mail

パスワードが変更できたら、姓名読み登録作業を行う。ここで登録した姓名読みは、後に交付されるメールアドレスのユーザ名となるので慎重に行うこと。 間違えてしまった場合演習室で変更ができず、とても面倒なことになる。この作業が終わったら、Tokyo Tech Web Mailが利用可能となり、 メールアドレスが交付される。詳しい使い方は次回説明される。

ふたつのポータルと教育用計算機システム

東工大は計算機システムを有している。スパコンTSUBAMEも計算機システムの一部である。 教育用計算機システムは教育用に学生向けに開放されている計算機システムの一部であり、この授業では教育用計算機システムの利用を開始する方法を説明する。 授業中に利用したMacも教育用電子計算機システムの一環として提供されており、本格的な利用を開始するにはアカウントの作成が必要である。

東工大ポータルとTSUBAMEポータル

東工大ポータルは東工大の提供する全てのWebサービスの入り口である。一方で、TSUBAMEポータルは教育用電子計算機システムの利用者向けの入り口である。 TSUBAMEポータルへは、東工大ポータルからアクセスすることが出来る。教育用計算機システムの利用開始にあたって、TSUBAMEポータルにアクセスする。

電子メールの宛先

電子メールには3つ宛先を指定する枠が存在する。それぞれ利用使途が異なるので適切な使い分けが求められている。

To

Toはメールの送信先の中でも、特に会話を投げかけている相手を指定する枠である。この枠に入れた相手に返信を期待している場合、Toを利用すると良い。

Cc

Cc(Carbon Copy)は送信先の中でも、会話を聞いて欲しい相手を指定する枠である。 この枠に入れた相手には返信を期待していないが、一体どのような会話が交わされているか認識して欲しい際に用いる。 Ccに指定した送信先は、会話に参加している他の人からでも見ることができる。

Bcc

Bcc(Blind Carbon Copy)はCcと同じように用いられる。但し、Bccに指定した送信先は会話の他の参加者からは見ることができない。 これは主に個人情報保護の観点から用いるものだ。例えば、サービスの提供者が顧客に対し一斉送信でメールを送信する場合は、顧客がお互いのアドレスを確認できる状態は好ましくないので、顧客全員をBccに指定する。 一部のメーラーではBccにだけ宛先が指定されており、Toが空欄の状態だとエラーが出て送信できない場合があるが、その時は自分自身をToに追加すると良い。

メールを支えるプロトコル

メールは一つの巨大なシステムに見えるが、実はメールの送信にまつわるシステムと受信にまつわるシステムは異なる。それぞれのシステムを簡単に紹介しよう。

SMTP

SMTP(Simple Mail Transfer Protocol)は、メールの送信を担当するプロトコルだ。メールクライアントからメールサーバへの送信や、メールサーバ間での送信に用いられている。 他にも、メール送信者が身分を偽造できないような認証の方法等も規定している。メールソフトにメールアドレスを設定した後、メールは受信できるが送信に失敗する場合は、SMTPの設定を見直そう。 (筆者はSMTPの設定の不備で送信機能だけ使えなかったことがある。)

POP

POP(Post Office Protocol)は、SMTPとは異なり受信を担当するプロトコルだ。POPでは、メールサーバに問い合わせを行い、届いていたメールをメールクライアントに移動させる。 そのため、メールクライアントに同期されたメールはサーバから削除されてしまう。複数の端末で利用している場合は、どれか一つの端末にメールが同期されることとなる。

IMAP

POPでは複数端末からの利用が面倒である。そんな時に使えるのがIMAP(Internet Message Access Protocol)である。IMAPも受信を行うプロトコルであることには変わりないが、 メールを同期した際に原本をサーバ上にも残しており、複数の端末から何度でも同期を行うことができる。メールの削除をしたい場合は、別途サーバに対して削除の指令を行う。

インターネットとウェブ

インターネットの概念を学ぶと共に、インターネット上で提供される代表的なシステムであるワールド・ワイド・ウェブについてその仕組みを学習する。

    目次

    インターネット IPアドレスとDNS WWW Web技術の変遷
インターネット

インターネットとは、インターナショナルなネットワークのことである。つまり、各組織の持つネットワークを、国際的に連結して一つのネットワークとして取り扱えるようにしたものである。インターネットは、様々な事業者が協調して支えており、たまに特定の企業がやらかしたせいでインターネットに大規模な障害が発生したというニュースが流れる。

IPアドレスとDNS

インターネットはインターネット・プロトコル(IP)という一種の取り決めによって実現されている。 その取り決めの中では、各ノードはIPアドレスという識別番号を与えられている。IPにはバージョンがいくつかあり、アドレスはバージョン4では1.234.56.78といった形式、バージョン6では2001:0db8:bd05:01d2:288a:1fc0:0001:10eeといった形式である。 しかしながら、このような数字は人間にとっては認識しにくいため、ドメインという別の識別記号を与えることが可能である。ドメインはピリオドによって区切られた文字列であり、区切られた文字列の内より後ろにある文字列がより大きな組織を表している。そして、ある組織に属するドメインを取得するには、その組織の認証を得ないといけない。 ちなみに、区切られた文字列の内最後の文字列はトップレベルドメインと呼ばれ、特定の国を指す記号(jp)や一般的な組織を表す記号(net)等が存在している。ドメインは、DNSというサービスによってIPアドレスに変換することが可能である。

WWW

上記のドメイン(IPアドレス)を用いると、世界中の情報リソースを一意に特定する文字列を規定することができる。それがURL(Uniform Resource Locator)である。 URLは通信に使用するプロトコルを指定するスキーム部、アクセス対象となるサーバを指定するホスト部、サーバ内における情報の位置を指定するパス部にわかれている。 通信に使用するプロトコルとして最も有名なものがHTTP(Hyper Text Transfer Protocol)/HTTPS(~Secure)である。 HTTPは主としてHTML(Hyper Text Markup Language)ファイルを伝送する。HTMLは文章に対して機械に認識可能な意味づけと構造化を与える書式である。 (ちなみに第六回の講義ノートまで書き上げた筆者にとっては、HTMLはもうあまり見たくない存在となりかけている。)

Web技術の変遷

平成に生まれた人間はなかなか認識しにくい事実であるが、インターネットという技術自体が比較的若い技術であり、未だ成長をしている。 Webに関して言っても、昔はWebサーバというと単に静的なファイルをホストしていただけなのだが、今となっては動的なコンテンツを生成・配信するに到っている。 要するに何が言いたいかと言うと、さっきの章で説明したHTMLなんかも、将来には使われなくなった枯れた技術になっている可能性は十分にあると言うことだ。 (筆者はWeb界隈の流行の移り変わりの速さにうんざりしたので、Webをあまり触っていない。)

情報倫理とセキュリティ

情報セキュリティとは何か学習し、インシデントを防ぐ力を身につける。

    目次

    キーコンセプト リスクマネジメント 攻撃者の心理 典型的攻撃手法
キーコンセプト

情報セキュリティは、情報が機密性・完全性・可用性・否認防止の4要素を備えていることである。機密性とは、情報を不適切な人間が閲覧することのないように適切な暗号化を施すことである。完全性とは、情報の伝送過程で情報の一部が欠損しないということである。また、可用性は情報を利用したいときに利用できる性質であり、サーバ等が停止せず稼働し続けられるということである。最後に、否認防止とは情報が本物であるということが確認することのできる状態を指し、後からでもその妥当性を検証できるということである。

リスクマネジメント

情報セキュリティに絶対は存在しない。現在のセキュリティの根幹にあるRSA暗号も、素因数分解の困難性に依存しているだけであり、計算機の性能向上により容易に破ることができる時代が到来する可能性は十分にある。重要なのは、セキュリティが破られることを想定して、二重三重の対策を施しておくことである。

攻撃者の心理
破壊

攻撃を行う物によっては、情報システムを破壊することに快感を覚える。特定の企業に恨みを持って居る場合や、損害を与えたい場合なども、破壊を目的とした攻撃が行われる。

情報収集

インターネットの世界では、手に入りにくい情報ほど高値がつく。あなたの名前や住所、メールアドレスやショッピングサイトの購買履歴などは攻撃者の格好の的である。また、クレジットカードの番号が盗まれてしまったが最後、攻撃者にお金を盗まれてしまう結果になるだろう。

リソースの盗用

あなたのパソコンの持つ処理能力や、あなたのメールアドレスそのものを目当てに攻撃を行う者も居る。例えば、ある攻撃者は自分の乗っ取ったコンピュータをスパムメールを送信することに使っていたという。他にも、乗っ取ったコンピュータで暗号通貨の採掘を行っている場合もある。

自己陶酔

時によっては、上記のどれにも当てはまらないのに攻撃を行う者も居る。そういう者達は大抵、強固な銀行のセキュリティを破ることを一種の実力証明とみなし、そういった実力を持っている自分に陶酔しているのだ。勿論、合法的に仕事として攻撃を行うホワイトハッカーという存在もあるが...。

典型的攻撃手法
ここではいくつかの典型的な攻撃手法を紹介する。ここで紹介するのは、データの送受信者以外の第三者がデータの盗聴や、なりすましを行える場合である。
リプレイ攻撃

ユーザ認証においては、ある所定の決められた手順に従って、サーバとクライアントの間で特定のデータが送受信される。サーバはクライアントをクライアントから送られてきたデータによって認証する。そこで、悪意のある第三者がやりとりされたデータを記録してしまい、あなたを騙ってサーバにあなたと同一の認証情報を送ってしまうことがある。そうすることで、サーバはあなたとあなたを騙る第三者の見分けがつかなくなり、あなたのアカウントが乗っ取られてしまうのだ。

中間者攻撃

ユーザがサーバと通信する際には、通信路にデータを流さねばならない。そういった通信路は、大抵の場合第三者が運営しているネットワークを経由しており、第三者が通信路に流れているデータを覗き見ることが可能である。このような攻撃は、公開鍵暗号を用いた暗号化(HTTPS等)を用いることで防ぐことができる。

セッションハイジャック

多くのユーザ認証が必要なウェブサイトでは、ユーザは認証を完了した証としてセッションIDを保持する。なんらかのアクションを行う際に、セッションIDを送信することでユーザが身分を証明するのだ。しかし、このセッションIDが悪意のあるスクリプトなどによって第三者に流失してしまった場合、第三者があなたのセッションIDを使ってあなたになりすますことが可能となる。XSS攻撃と組み合わせて行われることが多い。
